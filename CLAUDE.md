# KedoKudo profile repo

GitHub profile README for Chen (KedoKudo), generated by an automated pipeline — never edit `README.md` directly; edit `templates/README.md.tmpl` instead.

## Architecture

- `templates/README.md.tmpl` — `string.Template` source for the README (`$variable` placeholders)
- `scripts/update_profile.py` — fetches GitHub stats, writes `data/stats.json` + `assets/stats.svg`, renders `README.md`
- `scripts/ai_highlight.py` — sends recent public GitHub activity to an LLM via OpenRouter (default `anthropic/claude-opus-4.8`, override with `OPENROUTER_MODEL`) and writes `data/highlight.md`, which the renderer inserts into the "Currently Exploring" section
- `.github/workflows/update-profile.yml` — daily stats refresh; weekly (Monday) AI highlight refresh; commits the results

## Commands

Scripts are PEP 723 single-file scripts — run them with uv, no venv or install step:

```sh
# Full live run (set GITHUB_TOKEN to avoid rate limits)
uv run scripts/update_profile.py

# Re-render README/SVG from cached stats without hitting the GitHub API
STATS_SOURCE_FILE=data/stats.json uv run scripts/update_profile.py

# Refresh the AI highlight (needs OPENROUTER_API_KEY; no-ops gracefully without it)
uv run scripts/ai_highlight.py
```

## Notes

- The workflow needs an `OPENROUTER_API_KEY` repository secret for the AI highlight step; everything else runs on the default `GITHUB_TOKEN`.
- `data/highlight.md` is committed so the README renders the last good highlight even when the AI step is skipped.
